\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T, uint8\+\_\+t ROWS, uint8\+\_\+t COLUMNS$>$\newline
class modm::\+\+Matrix$<$ T, ROWS, COLUMNS $>$}
Class for handling common matrix operations. 

Having the width and height as template parameters has several advantages over the tradition dynamic matrix class:\+


\begin{DoxyItemize}
\item The compiler knows how many elements you have in your matrix and can unroll and optimize loops
\item You can ensure that you are not doing operations on matrices with incompatible sizes (multiplication for example). The compiler will tell you at compile time if you do.
\item When you receive a matrix as a function parameter, you don\textquotesingle{}t need to check that it is what you expect it to be. For example, if your function expects a 4x4 matrix, you\textquotesingle{}ll ask for a \doxylink{classmodm_1_1_matrix}{Matrix} and you are guaranteed to get what you asked for.
\end{DoxyItemize}
