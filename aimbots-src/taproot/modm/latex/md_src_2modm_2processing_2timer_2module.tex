This module provides polling based software timers for executing code after a delay or periodically in millisecond resolution via {\ttfamily modm::\+\+Clock} and in microsecond resolution via {\ttfamily modm::\+\+Precise\+Clock}.

To delay or delegate execution to the future, you can use {\ttfamily modm::\+\+Timeout} to set a duration after which the timeout expires and executes your code:\+

!!! warning "{}\+DO NOT use for hard real time systems!"{} You are responsible for polling these timers {\ttfamily execute()} methods as often as required. If you need to meed hard real time deadlines these are not the timers you are looking for!

!!! note "{}\+Timers are stopped by default!"{} If you want to start a timer at construction time, give the constructor a duration. Duration Zero will expire the timer immediately\hypertarget{md_src_2modm_2processing_2timer_2module_autotoc_md23}{}\doxysection{\texorpdfstring{Resolution}{Resolution}}\label{md_src_2modm_2processing_2timer_2module_autotoc_md23}
Two timer resolutions are available, using {\ttfamily modm::\+\+Clock} for milliseconds and {\ttfamily modm::\+\+Precise\+Clock} for microseconds. They follow the same naming scheme:\+


\begin{DoxyItemize}
\item {\ttfamily modm::\+\+Timeout}, {\ttfamily modm::\+\+Periodic\+Timer}:\+ 49 days in milliseconds and 8 bytes.
\item {\ttfamily modm::\+\+Precise\+Timeout}, {\ttfamily modm::\+\+Precise\+Periodic\+Timer}:\+ 71 mins in microseconds and 8 bytes.
\end{DoxyItemize}

If you deal with short time periods, you can save a little memory by using the 16-\/bit versions of the same timers:\+


\begin{DoxyItemize}
\item {\ttfamily modm::\+\+Short\+Timeout}, {\ttfamily modm::\+\+Short\+Periodic\+Timer}:\+ 65 seconds in milliseconds and 4 bytes.
\item {\ttfamily modm::\+\+Short\+Precise\+Timeout}, {\ttfamily modm::\+\+Short\+Precise\+Periodic\+Timer}:\+ 65 milliseconds in microseconds and 4 bytes. 
\end{DoxyItemize}
