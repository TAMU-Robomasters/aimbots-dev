\doxysubsection{Detailed Description}
Counting semaphore. 

This module implements counting semaphores on top of protothreads. Semaphores are a synchronization primitive that provide two operations:\+ \doxylink{classmodm_1_1pt_1_1_semaphore_a08448026d80df889cf78d17250526f89}{acquire()} and \doxylink{classmodm_1_1pt_1_1_semaphore_a4ad53508e0f6bb5a68e4e6b41782bad0}{release()}. Sometimes these operations are also called {\ttfamily wait} and {\ttfamily signal}. The \doxylink{classmodm_1_1pt_1_1_semaphore_a08448026d80df889cf78d17250526f89}{acquire()} operation checks the semaphore counter and "{}blocks"{} the thread if the counter is zero. The \doxylink{classmodm_1_1pt_1_1_semaphore_a4ad53508e0f6bb5a68e4e6b41782bad0}{release()} operation increases the semaphore counter but does not block. If another thread has blocked waiting for the semaphore that is signalled, the blocked thread will become runnable again.

This class can for example be used for resource limiting or credit tracking:\+

Resource limit:\+
\begin{DoxyItemize}
\item {\ttfamily initial} = number of resources
\item \doxylink{classmodm_1_1pt_1_1_semaphore_a08448026d80df889cf78d17250526f89}{acquire()} to gain access to a resource
\item \doxylink{classmodm_1_1pt_1_1_semaphore_a4ad53508e0f6bb5a68e4e6b41782bad0}{release()} to release the resource
\end{DoxyItemize}

Credit tracking:\+
\begin{DoxyItemize}
\item {\ttfamily initial} = 0
\item \doxylink{classmodm_1_1pt_1_1_semaphore_a08448026d80df889cf78d17250526f89}{acquire()} to increment (credit)
\item \doxylink{classmodm_1_1pt_1_1_semaphore_a4ad53508e0f6bb5a68e4e6b41782bad0}{release()} to consume (debit)
\end{DoxyItemize}

As protothreads implement a form cooperative multithreading no synchronisation is needed for accessing global variables or for critical sections. Just don\textquotesingle{}t use PT\+\_\+\+YIELD() or PT\+\_\+\+WAIT\+\_\+\+UNTIL() inside a critical sections and you\textquotesingle{}re fine.

Therefore there\textquotesingle{}s no Mutex implementation, it isn\textquotesingle{}t needed.

Acquire the semaphore. 

The "{}acquire"{} or "{}wait"{} operation causes the protothread to block while the counter is zero. When the counter reaches a value larger than zero, the protothread will continue.

Use this always within a PT\+\_\+\+WAIT\+\_\+\+UNITL() macro inside a modm::\+pt::\+\+Thread::\+run() method.
