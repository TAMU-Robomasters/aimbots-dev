\doxysubsection{Detailed Description}
Dummy implementation of an I/\+O pin.

This class can be used when a pin is not required. All functions are dummy functions which do nothing. {\ttfamily read()} will always return {\ttfamily false}.

For example when creating a software SPI with the modm::\+\+Software\+Simple\+Spi class and the return channel (MISO -\/ Master In Slave Out) is not needed, a good way is to use this class as a parameter when defining the SPI class.

Example:\+ 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#include\ <modm/architecture/platform.hpp>}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{namespace\ }pin}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ \textcolor{keyword}{typedef}\ GpioOutputD7\ Clk;}
\DoxyCodeLine{\ \ \ \ \textcolor{keyword}{typedef}\ GpioOutputD5\ Mosi;}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{modm::SoftwareSpiMaster<\ pin::Clk,\ pin::Mosi,\ GpioUnused\ >\ Spi;}
\DoxyCodeLine{}
\DoxyCodeLine{...}
\DoxyCodeLine{Spi::write(0xaa);}

\end{DoxyCode}
