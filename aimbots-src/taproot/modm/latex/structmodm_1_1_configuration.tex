\subsubsection*{template$<$typename Parent, typename Enum, typename Parent::\+\+Underlying\+Type Mask, typename Parent::\+\+Underlying\+Type Position = 0$>$\newline
struct modm::\+\+Configuration$<$ Parent, Enum, Mask, Position $>$}
Class for accessing a configuration in a register.

A configuration is a combination of bits, whose meaning is different from their numerical value. A typical example is a 2-\/bit prescaler with the values \+[0,1,2,3]\+ correspond to the meanings \+[Div1, Div2, Div4, Div8]\+.

The \doxylink{structmodm_1_1_configuration}{Configuration} class belongs to a specific \doxylink{structmodm_1_1_flags}{Flags} class and connects a corresponding Sascha Schade enum with a specific mask and bit position. This means that the \doxylink{structmodm_1_1_configuration}{Configuration} class will mask and shift the values whenever needed. However, shifting is only done when the Position template parameter is non-\/zero.

To understand when shifting is necessary, consider the following two scenarios:\+
\begin{DoxyEnumerate}
\item There is only one prescaler configuration at bit position 4, and
\item There are two or more prescaler configurations at several bit positions.
\end{DoxyEnumerate}

For the first scenario, no shifting is required, as we can declare the prescaler values already in the shifted position:\+
